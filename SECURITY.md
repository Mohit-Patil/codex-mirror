# Security Policy

## Supported versions

Only the latest release line is supported for security fixes.

## Reporting a vulnerability

Please do not file public GitHub issues for sensitive vulnerabilities.

Use private disclosure:

1. Open a private security advisory in GitHub for this repository, or
2. Contact maintainers privately with:
   - impact summary
   - reproduction steps
   - affected versions
   - potential mitigation

We will acknowledge reports promptly and coordinate a fix + disclosure timeline.
